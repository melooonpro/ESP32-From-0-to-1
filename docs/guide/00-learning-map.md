# 学习地图

本教程以 ESP32-S3 为例。你不需要先成为 C 语言高手，也不需要先读完芯片手册；更好的路线是：先让板子跑起来，再用一个个可观察的小现象理解系统。

## 推荐学习节奏

第一周只追求“能跑”：安装环境、烧录 Hello World、看懂串口输出。第二周开始碰外设：按键、LED、PWM、I2C 扫描。第三周进入 Wi-Fi 和 HTTP，让设备和网络产生关系。第四周再系统学习 FreeRTOS，因为这时你已经见过“为什么需要多个任务”。

::: warning 不要一开始就追求大项目
初学者最常见的挫败感不是“不会”，而是同时处理太多未知：硬件接线、驱动 API、CMake、RTOS、网络、云平台。每章只增加一两个新概念，才是最快的路线。
:::

## 本站如何维护

每章遵循同一种结构：先讲直觉，再列准备工作，然后给出最小例程，最后给出 AI 提示词。示例代码放在仓库 `examples/` 下，正文只摘出关键片段，避免教程变成代码墙。

后续迭代建议按“能被学生独立复现”的标准合并内容：每新增一个外设或应用，至少包含硬件连接、完整工程、现象验证、常见错误、AI 提示词。
